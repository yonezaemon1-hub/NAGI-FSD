# Negative Results Log — 2026-05-30

単一GPU (GTX 1070) で「GPU実行タイミングに検知可能な構造があるか」を、
一晩かけて反証ベースで掘った記録。すべて対照実験 (シャッフル / 純粋乱数置換 /
seed振り / 入力定数化) で検証している。結論から言うと、**単一GPUのmatmul実行時間
という対象には、捉えるべき時間構造が存在しなかった。** 5段階で確定させた。

環境: GTX 1070 / Python 3.10 / PyTorch 2.7.1+cu118 / matmul 2048² / N=200

---

## 反証1: V(t) スパイクはアーティファクト

`nagi_control.py`

- 生 step_time は「スパイク領域」(step 120–132) でも ~3.74ms で平坦。異常は体積変換後にのみ出現。
- V(t) の四面体は (step_time, sync_drift, cpu_launch) の3量を軸にしていた。
  cpu_launch を定数化するとスパイクは **0.0×** に消滅 → 原因は `torch.randn` のCPU側確保ジッタ。
- 5 seed で V(t) ピーク位置は step 168/48/196/0/1 (std ≈ 83)。step 125 固定ではない。

## 反証2: PI は順位ノイズ (順序構造ではない)

`pi_control.py`

- 通常PI = 0.915 / シャッフルPI = 0.925 / 純粋乱数PI = 0.910 → 統計的に同一。
- 順序を壊してもPI不変、中身を乱数に置換してもPI不変。
  → PIは「ほぼ同一の約10値に順位をつけたときのランクノイズ」を測っているだけ。
- 「クロスサイズで一定のPI」は現象の頑健さではなく、アーティファクトがサイズ非依存なだけ。

## 反証3: 計測精度の問題ではない

`event_control.py`

- wall-clock を CUDA event (GPU内カーネル実時間) に置換しても、
  通常PI ≒ シャッフル ≒ 乱数 (差 0.005–0.030)。
- 計測精度を上げても構造は出ない → 問題は精度ではなく対象。

## 反証4: 依存構造を作っても効かない

`dependency_test.py`

- 独立処理 vs 逐次依存処理 (前の出力を次の入力に) で比較。
- 依存処理でも 通常PI 0.920 / シャッフル 0.880 / 乱数 0.910 (差 0.04)。
- 依存を作ってもPIは反応せず。問題は対象の依存有無ですらない。

## 反証5: 実行時間そのものに時間構造が無い

`successor_search.py`

- PI / 自己相関lag1 / CV変化 の3量を独立・依存の両方で評価。
- 自己相関 lag1 通常値: 独立 = +0.011 / 依存 = −0.002 → どちらもほぼゼロ。
- 前ステップの実行時間は次ステップを予測しない。
  → 2048² matmul は中身に依らず同FLOPS・同時間。
     「計算グラフの依存」と「実行時間の系列構造」は別物だった。
     GPU実行時間は、計算依存があっても時間的に独立。

## 結論

単一GPUのmatmul実行時間という井戸は、4つの独立な方法で掘って水が無いことを確定させた。
PI / V(t) / 自己相関 / CV変化 のいずれも、本物の構造には反応しない。
これらが捉えていたのは、すべて「ほぼ一定の安定値に微小差増幅をかけたときのノイズ」だった。

## 消去法が指す次の場所

今夜潰した4つの井戸は、いずれも「複数実行単位の待ち合いが無い場所」だった。
単一カーネルには待ち合いが無く、だから実行時間が時間的に独立した。
構造が存在しうるのは、**複数の実行単位が互いの完了を待つ場所** — すなわち
NCCL collective (all_reduce 等)。silent NCCL stall はその待ち合いが静かに崩れる現象。
待ち合いがある = 実行時間に依存が生まれる = 時間構造が存在しうる。

ネガティブな結果4つの消去法が、次に掘るべき場所を NCCL collective へ論理的に絞り込んだ。
ただしこれは最低でもマルチGPU環境を要するため、単一GPUでの検証はここで打ち止め。

## 補足: 単一GPUで残る最後の角度

matmul は計算律速で実行時間が中身に依らなかった。メモリ帯域律速な処理
(巨大テンソルのコピー等) は確保状態に依存して時間が揺れる可能性があり、
単一GPUで「待ち」がありうる唯一の場所。次の実験対象。
